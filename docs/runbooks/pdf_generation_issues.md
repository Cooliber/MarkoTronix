# PDF Generation Issues Runbook

This runbook provides guidance on handling PDF generation issues in the HVAC CRM system.

## Overview

The offer generation service generates PDF files for offers. This runbook covers procedures for handling PDF generation issues and recovery.

## Monitoring PDF Generation Health

### Health Check Endpoints

The offer generation service exposes a health check endpoint that includes the status of the storage directory:

- Offer Generation Service: http://localhost:18002/health

### Prometheus Alerts

Prometheus is configured to alert when:
- PDF generation has a high 95th percentile latency (> 10s over 5 minutes)
- Template rendering has a high 95th percentile latency (> 5s over 5 minutes)

## Common PDF Generation Issues and Resolution

### PDF Generation Failures

**Symptoms**:
- Offer generation service logs PDF generation errors
- Offers are created but have no PDF path
- Prometheus alert for HighPDFGenerationLatency is triggered

**Resolution**:

1. Check if the storage directory is accessible:
   - Check the offer generation service health endpoint for storage status
   - Verify that the `STORAGE_PATH` environment variable is set correctly
   - Check if the directory exists and is writable:
     ```bash
     docker exec -it <offer-generation-container-id> ls -la $STORAGE_PATH
     ```

2. Check if the required libraries are installed:
   - Verify that WeasyPrint and Pyppeteer are installed:
     ```bash
     docker exec -it <offer-generation-container-id> pip list | grep -E 'weasyprint|pyppeteer'
     ```
   - If not installed, install them:
     ```bash
     docker exec -it <offer-generation-container-id> pip install weasyprint pyppeteer
     ```

3. Check if Chromium is installed (required for Pyppeteer):
   - Verify that Chromium is installed:
     ```bash
     docker exec -it <offer-generation-container-id> which chromium
     ```
   - If not installed, install it:
     ```bash
     docker exec -it <offer-generation-container-id> apt-get update && apt-get install -y chromium
     ```

4. Check if there are any errors in the HTML content:
   - Get the HTML content of the offer:
     ```bash
     curl http://localhost:18002/offers/<offer_id>/html
     ```
   - Check for any syntax errors or invalid HTML

5. Try regenerating the PDF:
   ```bash
   curl -X POST http://localhost:18002/offers/<offer_id>/regenerate-pdf
   ```

6. If Pyppeteer fails, try using WeasyPrint as a fallback:
   - Update the `generate_pdf_for_offer` function to use WeasyPrint instead of Pyppeteer
   - Restart the offer generation service

7. Verify that the PDF is generated by checking the offer:
   ```bash
   curl http://localhost:18002/offers/<offer_id>
   ```

### Template Rendering Failures

**Symptoms**:
- Offer generation service logs template rendering errors
- Offers are created but have no HTML content
- Prometheus alert for HighTemplateRenderingLatency is triggered

**Resolution**:

1. Check if the OpenAI API is available:
   - Check the offer generation service health endpoint for OpenAI status
   - Follow the [OpenAI API Issues Runbook](openai_api_issues.md) if necessary

2. Check if there are any errors in the template data:
   - Check the offer data for any missing or invalid fields
   - Verify that the client data is complete

3. Try regenerating the offer content:
   - Update the offer with valid data:
     ```bash
     curl -X PUT http://localhost:18002/offers/<offer_id> -H "Content-Type: application/json" -d '{
       "client_name": "Test Client",
       "client_email": "test@example.com",
       "client_phone": "123456789",
       "client_address": "Test Address",
       "service_type": "HVAC Installation",
       "description": "Test description",
       "price": 1000.0,
       "currency": "PLN",
       "valid_days": 30
     }'
     ```

4. If the OpenAI API is unavailable, use a pre-generated template:
   - Create a simple HTML template for offers
   - Update the `generate_offer_content` function to use the template when the OpenAI API is unavailable

5. Verify that the HTML content is generated by checking the offer:
   ```bash
   curl http://localhost:18002/offers/<offer_id>/html
   ```

### Storage Directory Issues

**Symptoms**:
- Offer generation service reports storage directory issues in health checks
- Offer generation service logs storage directory errors
- PDF generation fails with file system errors

**Resolution**:

1. Check if the storage directory exists:
   ```bash
   docker exec -it <offer-generation-container-id> ls -la $STORAGE_PATH
   ```

2. If the directory doesn't exist, create it:
   ```bash
   docker exec -it <offer-generation-container-id> mkdir -p $STORAGE_PATH
   ```

3. Check if the directory is writable:
   ```bash
   docker exec -it <offer-generation-container-id> touch $STORAGE_PATH/test.txt
   ```

4. If the directory is not writable, fix the permissions:
   ```bash
   docker exec -it <offer-generation-container-id> chmod -R 777 $STORAGE_PATH
   ```

5. Check if there's enough disk space:
   ```bash
   docker exec -it <offer-generation-container-id> df -h
   ```

6. If there's not enough disk space, free up some space or increase the disk space allocation.

7. Verify that the storage directory is accessible by checking the offer generation service health endpoint.

## Fallback Strategies

If PDF generation is failing, consider the following fallback strategies:

1. **Use WeasyPrint Instead of Pyppeteer**:
   - WeasyPrint is more reliable but has fewer features
   - Update the `generate_pdf_for_offer` function to use WeasyPrint instead of Pyppeteer

2. **Use Pre-Generated Templates**:
   - Store pre-generated templates for common offer types
   - Use these templates when the OpenAI API is unavailable

3. **Generate PDFs on Demand**:
   - Instead of generating PDFs in the background, generate them when requested
   - Add a "Generate PDF" button to the UI

4. **Use External PDF Generation Service**:
   - Use a service like DocRaptor or PDFShift
   - Implement a fallback to these services when local PDF generation fails

## Preventive Measures

To prevent PDF generation issues:

1. **Monitor Storage Usage**:
   - Set up alerts for low disk space
   - Implement automatic cleanup of old PDFs

2. **Implement Retry Mechanisms**:
   - Use exponential backoff for retries
   - Set appropriate retry limits

3. **Implement Fallback Mechanisms**:
   - Provide fallback functionality when PDF generation fails
   - Implement multiple PDF generation methods

4. **Implement Logging and Monitoring**:
   - Log all PDF generation attempts
   - Monitor PDF generation latency and success rate

5. **Implement Regular Testing**:
   - Regularly test PDF generation with different templates
   - Implement automated tests for PDF generation

## Escalation Path

If you are unable to resolve a PDF generation issue, escalate to the appropriate team:

1. **Level 1**: On-call engineer
2. **Level 2**: Offer generation team
3. **Level 3**: Engineering manager
4. **Level 4**: CTO
